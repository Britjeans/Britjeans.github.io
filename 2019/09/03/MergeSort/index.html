Leetcode 315 Count of Smaller Numbers After Self

2019-09-03
#Sort #Leetcode

After we did merge sort on the first part and second part of the array, all the elements in the second part still have greater indexes than the elements do in the first part. Now when we compare nums[i] and nums[j] (left <= i < mid <= j < right), if nums[j] is smaller than nums[i], then for all the indexes k <= j && k >= mid , we have nums[k] < nums[i], thus we can tell on the second half, we have j - mid smaller numbers for num[i]. However, after we did merge sort on both part, the indexes may not maintain its original status in both parts. So here I used pair<int, int> to store both the value and its original index.
