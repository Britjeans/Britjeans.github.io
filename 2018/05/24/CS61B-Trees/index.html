Tree Characteristics

  • The height of a node in a tree is the smallest distance to a leaf. That is, a leaf has height 0 and a non-empty tree’s height is one more than the maximum height of its children. The height of a tree is the height of its children. The height of a tree is the height of its root.
  • The depth of a node in a tree is the distance to the root of that tree. That is , in a tree whose root is R, R itself has depth O in R, and if nodes S!=R is in the tree with root R, then its depth is greater than its parents’.

Quadtrees

A quadtree is a tree data structure in which each internal node has exactly four children. Quadtrees are the two-dimensional analog of octrees and are most often used to partition and two-dimensional space by recursively subdividing it into four quadrants or regions.

Quadtrees may be classified according to the type of data they represent, including areas, points, lines and curves. Quadtrees may also be classified by whether the shape of the tree is independent of the orderin which data is processed. The following are common types of quadtrees.

  1. Region quadtree
  2. Point quadtree
  3. Point-region (PR) quadtree
  4. Edge quadtree
  5. Polygonal map (PM) quadtree

  6. Compressed quadtrees

    Original definition: a quadtree is either

  • Empty, or
  • An item at some position (x,y), called the root, plus
  • four quadtrees, each containing only items that are northwest, northeast, southwest, and southeast of (x,y).

Point- region (PR) Quadtrees

If we use a QuadTree to track moving objects, it may be useful to be able to delete items from a tree : when an object moves, the subtree that it goes in may change.

Navigating PR Quadtrees

If (x,y) is outside the bouding rectangle of T, or T is empty, then (x,y) is not in T.

Otherwise, if T contains a small set of items, then (x,y) is in T if it is among these itmes.

Otherwise, T consists of four quadtrees. Recursively look for (x,y) in each.

Alpha-Beta Pruning

Cutting off the Search

  • If you could traverse game tree to the bottom, you’d force a win (if it’s possible).
  • Sometimes possible near the end of game.
  • Unfortunately, game trees tend to be either infinite or impossibly large.
  • So, we choose a maximum depth, and use a heuristic value computed on the position alone as the value at that depth.
  • Or we might use iterative deepening and repeat the search at increasing depth until time is up.

Overall Search Algorithm

Depending on whose move it is (maximizing player or minimizing player), we’ll search for a move estimated to be optimal in one direction or the other.

Search will be exhaustive down to a particular depth in the game tree; below that, we guess values.

Pass alpha and beta as limits

High player does not care about exploring a position further once he knows its value is larger than what the minimizing player will never allow that position to come about.

Likewise, minimizing player won’t explore a position whose value is less than what the maximizing player knows he can get alpha.

  • To start, a maximizing player will find a move with

    findMax(current position, search depth -oo,+oo)

  • minimizing player

    findMin(current position, search depth -oo,+oo)
